# Common Conventions for Paper Pipeline Agents

## 1. 입력/출력 경로
- 모든 산출물은 `workspace/<slug>/` 하위에 저장.
- `<slug>`는 호출자 prompt에서 추출 (소문자, 영숫자+하이픈). 없으면 사용자에게 한 줄 질문.
- 기존 파일 덮어쓰기 전에는 한 줄 경고 후 진행.

## 2. 환각 방지 (절대 규칙)
- 확인되지 않은 저자/논문/수치/URL **인용 금지**. 의심스러우면 일반 표현으로.
- WebSearch 결과의 제목/연도는 그대로 보존. 추측 보강 금지.
- 04_experiments/results.md가 MOCK이면 그 이후 모든 단계는 본문에 "(preliminary, pending full evaluation)" 단서를 유지.

## 3. 비용 절감 (호출자/하위 agent 공통)
- 이전 단계 산출물을 prompt에 통째로 붙이지 말 것. 항상 파일 경로 전달 → Read.
- 같은 파일을 한 작업 내에서 두 번 Read 금지 (한 번 읽고 메모리에 유지).
- Glob/Grep 우선, 큰 디렉토리 전체 ls 지양.

## 4. 언어
- 본문: 호출자 prompt에 언어 지정이 있으면 그대로. 없으면 한국어.
- 단, `paper-writer`만 예외 — 학회 제출 가정 영어 default.
- 기술 용어는 한국어+영문 병기 (예: "확산 모델(diffusion model)").

## 5. 출력 길이 가이드
- 각 agent의 본문에 명시된 권장 단어수 준수.
- 길이가 권장의 1.5배 초과 또는 50% 미달이면 자체 검토 후 조정.

## 6. 최종 보고 톤
- 작업 종료 시 응답은 **1-3문장**. 산출물 경로 + 핵심 한 줄 + (있다면) 경고/권고.
- 다음 단계 권유 금지 (오케스트레이터가 결정).
- "분석을 진행하겠습니다", "다음으로 ..." 같은 메타 발언 금지.

## 7. 에러 시 동작
- 입력 파일 부재 → 작성 중단, 호출자에게 보고 ("입력 파일 X 부재, 단계 N부터 재실행 필요").
- 도구 실패 → 자동 재시도 1회까지. 그 이상 실패 시 보고.
- 사용자 입력이 필요하면 작업 시작 전에 한 줄 질문. 진행 중 중단은 마지막 수단.

## 8. 모델 escalation
- 본 agent의 frontmatter `model:` 필드를 따름.
- sonnet이 같은 입력에 두 번 실패한 경우 호출자에게 "opus 재시도?" 한 줄 질문.
