# method.md — Method 섹션 작성 템플릿 (재사용)

> 용도: AI/ML 논문 Method 섹션(보통 Section IV) 작성 시 에이전트에 주입하는 범용 가이드.
> 아래 구조·규칙을 따르되, 해당 논문의 문제·표기에 맞춰 내용을 채운다.

## STEP 0 — 논문 유형 판별 (구조가 갈림)
- **(A) 새 방법 제안형**: 새 모델/알고리즘/손실/구조 → 핵심을 구성요소로 분해해 서술.
- **(B) 분석·실증형**: 새 방법 없이 현상 분석/측정 → method = "분석 프로토콜·측정 절차".
→ 첫 문장에서 어느 쪽인지 독자에게 명확히 한다.

## 필수 구조
1. **Overview (1문단)**: 무엇을 + 왜. 핵심 아이디어/직관 한 문단 (수식 최소).
2. **Formulation & Notation**: 문제를 형식화하고 기호 정의. 별도 Preliminaries가 있으면 그 기호를 재사용(중복 정의 금지).
3. **Core Method / Protocol**:
   - (A형) 핵심 구성요소를 하위절로. 각 요소 = 정의 → 수식/동작 → 설계 이유(why).
   - (B형) 측정 대상 + metric 정식화 + 실험 조건 설계 (+가설이 있으면 가설별 예측).
4. **Algorithm / Procedure** (해당 시): 단계 또는 의사코드. 입력→출력 명확히.
5. **Design Rationale**: 주요 선택의 근거를 1~2줄씩. 대안 대비 왜 이걸 택했는지.

## HARD RULES
- **표기 일관성**: 기호는 첫 등장 시 정의, 이후 동일 사용. 앞 섹션과 충돌 금지.
- **결과 금지**: 정량 수치/성능은 전부 Experiments로. 여기선 "어떻게"만 쓴다.
- **중복 금지**: Related Work·Preliminaries 설명을 반복하지 말고 짧게 참조.
- **그림**: pipeline overview figure 1개 권장. 캡션은 자기완결적으로.
- **톤**: 단정보다 정확. 과장·홍보 표현 배제.
